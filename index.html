Навыки

    Работа с различными ОС:
    • Windows: cmd, powershell, администрирование системы. Настройка брандмауэра, path, реестра и пр.
    • Linux: ubuntu, mint. Настройка системы под себя, установка утилит (cat, tail, nano и пр.), использование команд (cd, rm, ls, touch и пр.)
    • Android: Управление системой на уровне опытного пользователя. Опыт тестирования и отладки мобильных приложений (ADB, Android Studio)
    Теория тестирования:
    • Знание и понимание видов и принципов тестирования ПО
    • Понимание классификации тестирования ПО, умение применять различные техники тест-дизайна и методологий тестирования ПО
    • Знание процессов, методологий и жизненных циклов тестирования и разработки ПО (STLC, SDLC)
    • Понимание концепции и подходов к автоматизации тестирования
    Практика тестирования:
    • Проведение различных видов тестирования ПО (функциональное, юзабилити, регрессионное, нагрузочное, кросс-браузерное и другие)
    • Участие в тестировании на различных этапах разработки (модульное, интеграционное, системное, приемочное)
    • Тестирование функционала и бизнес-логики веб, мобильных и десктоп приложений
    • Тестирование UI/UX для веб, мобильных и десктоп приложений
    • Тестирование верстки и адаптивности страниц и экранов на соответствие макетам
    • Тестирование API
    • Исследование причин, выявление сбоев и локализация дефектов
    • Постановка задач на исправление и доработку функционала, а также контроль их выполнения
    • Опыт работы единственным тестировщиком в команде
    • Опыт тестирования безопасности веб, мобильных и десктоп приложений
    • Опыт тестирования локализации
    Инструменты тестирования:
    • Тестирование API и интеграций: Postman, Swagger, SoapUI, DevTools, Сurl
    • Опыт работы с данными и системами мониторинга: Charles, Fiddler, Zabbix, Kibana, ELK
    • Подключение к удаленным Windows и Unix хостам, сбор и анализ логов: Putty, SSH, RDP, Telnet, WinSCP
    • Инструменты логирования: Kibana.
    • Работа с файлами логов и конфигами: ADB, Android Studio (Logcat), Notepad++, Sublime text, DevTools, gedit, cat, nano
    • Опыт визуализации данных с помощью Kibana, Grafana, самописных продуктов (json в графический интерфейс)
    • Опыт работы с моками и заглушками на базе инструментов: Postman, SOAP UI
    • Утилиты администрирования очередей: Rabbit MQ Admin
    • Инструменты взаимодействия с СУБД: dbeaver, MongoDB
    • Работа с системой контроля версий: Git, GitHub, GitLab
    • Работа с эмуляторами для мобильных приложений и отладка: ADB, Android Studio
    • Базовые знания проведения нагрузочного тестирования с Jmeter
    Работа с документацией:
    • Разработка и поддержка полного цикла тестовой документации (тест планы, тест кейсы, чек листы, отчеты о проведении тестирования и др.)
    • Разработка и актуализация технической документации (сценарии использования, фрагменты технических заданий и спецификаций), в том числе при взаимодействии с заказчиком
    • Анализ требований, технических заданий, технической документации и макетов
    • Опыт написания пользовательских инструкций, обучение и сопровождение пользователей
    • Ведение баз знаний GitLab Wiki, Confluence.
    • Ведение баг-репортов в redmine, jira,
    • Работа с артефактами приемочного тестирования (ПСИ и ПМИ)
    Системы управления тестированием и документацией:
    • Ведение проекта, организация работы, тестовая документация и баг-трекинг: Redmine, GitLab, GitHub, Jira, TestIT, TestRail, Confluence, доски Miro, Figma, Bitrix24, Microsoft Teams и др.
    Опыт программирования и верстки:
    • Хорошие знания Python: (синтаксис, алгоритмика, автоматизация, приложения, боты). Примеры: Архив заданий python | test | daily-byte daily tasks | test
    • Хорошие знания JavaScript (синтаксис, алгоритмика, приложения, автоматизация) | JS weather bot
    • Базовые знания Java (синтаксис, алгоритмика, приложения, автоматизация) Примеры автоматизации
    • Базовые знания вёрстки (HTML, CSS, JavaScript)
    • Базовые знания C# (синтаксис, алгоритмика, приложения)
    • Базовые знания PHP (синтаксис)
    Автоматизация тестирования:
    • Запуск автотестов. Анализ тест-репорта.
    • Написание UI автотестов на Playwright + java(+junit) + js + python Примеры
    • Мониторинг статусов и результатов прохождения авто-тестов через Allure
    • Автоматизация тестирования средствами Postman (коллекции)
    Сетевые технологии, протоколы и способы интеграции:
    • Понимание устройства сетевых протоколов и способов взаимодействия различных систем
    • Знания сетевого взаимодействия, сетевого оборудования, маршрутизации и модели OSI
    • Знания клиент-серверной архитектуры и способов интеграции между компонентами приложений (REST, SOAP, WebSocket, gRPC, GraphQL и другими)
    • Знания особенностей взаимодействия микро-сервисной архитектуры
    • Опыт работы с протоколами HTTP(S), IMAP, SMTP, FTP, SSH, SMB и другими
    • Анализ входящих сообщений в Rabbit MQ
    • Понимание форматов передачи данных (XML, JSON, YAML)
    Администрирование в ИТ:
    • Знания внутреннего устройства и архитектуры ПК
    • Практический опыт диагностики и ремонта компьютерной (и не только) техники
    Знания и опыт работы с базами данных и SQL:
    • Опыт работы с SQL базами данных (MySQL, OracleDB, PostgreSQL, MS SQL) написание SQL запросов средней сложности (CRUD, выборка, группировка, сортировка, объединения)
    • Опыт работы с NoSQL базами данных (MongoDB) написание запросов Regexp в MongoDB с целью аггрегации и поиска данных
    • Опыт работы с не реляционными базами данных (Elastic Search)
    Виртуализация:
    • Работа с системами виртуализации и контейнеризации (VirtualBox, Hyper-v, Docker, Kubernetes)
    • Опыт развертывания и администрирования приложений в контейнерах при помощи Docker
    • Работа с облачными сервисами, подключение к ВМ, выполнение команд через SSH или VNC
    Системы контроля версий и CI\CD:
    • Работа с Git (cmd/powershell, terminal, git-bash, desktop)
    • Знания о запуске пайплайна деплоя тестового окружения в jenkins

    • Опыт наставничества
    • Опыт проведения онбординга для QA специалистов
    • Планирование и контроль работы младших тестировщиков

    • Высокий уровень самоорганизации и самостоятельности
    • Ответственность, усидчивость, внимательность к мелочам
    • Системное мышление, аналитический склад ума. Умение разобраться в новой системе или в кратчайшие сроки.

Прошлые проекты

Проект облачных технологий

Длительность проекта: Октябрь 2023 – Февраль 2025 (1 год 4 месяца)

Коробочное решение для управления облачными вычислительными мощностями внутри предприятия
Инструменты: TestIT, Postman, DevTools, Fiddler, Swager, Putty, RabbitMQ Admin, Playwright
Технологический стек: Ruby, Python, JS, RabbitMQ, Kubernetes

    Основные обязанности:
  • Тестирование требований по документации
  • Подготовка и актуализация тестовой документации
  • Проведение необходимых функциональных и нефункциональных видов тестирования
  • Тестирование существующих и новых программных интерфейсов, интеграций и соответствующей бизнес логики
  • Тестирование удаленных виртуальных серверов через Putty (ssh)
  • Анализ логов, расположенных напрямую на удаленных серверах (k8s-pod-shell) как в реальном времени, так и за определенный период с фильтрацией
  • Регулярное проведение регресса и интеграционного тестирования
  • Постановка задач на устранение дефектов
    Достижения:
  • Провел частичную автоматизацию создания тестовых сущностей с помощью JavaScript + playwright на UI
  • Выполнено тестирование нового функционала отложенных действий, применяемых к продукту, при отсутствии актуальной документации

Веб-конструктор страховых продуктов

Длительность проекта: Июль 2021 – Октябрь 2023 (2 года 3 месяца)

Внутренний сервис для страховой компании по урегулированию убытков (комплекс веб приложений)
Инструменты: TestIT, Postman, DevTools, Fiddler, Swager, Putty, RabbitMQ Admin, Playwright
Технологический стек: Ruby, Python, JS, RabbitMQ, Kubernetes

    Основные обязанности:
  • Углубленное изучение матчасти и предметной области по страхованию
  • Работа с требованиями и спецификациями на новые и уже реализованные фичи, тестирование требований
  • Разработка и актуализация тестовой документации (кейсы, планы)
  • Ведение задач Jira, оформление дефектов и их дальнейший контроль
  • Проведение функционального и интеграционного тестирования сервисов в рамках работы над новым внедряемым функционалом приложения
  • Тестирование соответствия создаваемых системой документов заявленным шаблонам
  • Тестирование и верификация справочников и корректности передаваемой информации между сервисами в рамках интеграционного тестирования
  • Тестирование API (REST\SOAP), подготовка и отправка запросов через Postman и SoapUI для создания/корректировки данных
  • Подготовка и проведение демо нового функционала для заказчиков
  • Написание SQL скриптов для создания, корректировки и валидации данных в БД (select, join, update, insert, агрегатные и т.п.)
  • Анализ логов работы системы и отдельных компонентов
  • Коммуникация с командами смежных сервисов

Сервис по жилищно-коммунальному хозяйству (ЖКХ)

Длительность проекта: Апрель 2021 – Июль 2021 (4 месяца)

Комплекс веб-сервисов для учета и инспекции зданий в ЖКХ (веб приложения)
Инструменты: GitLab, Jira, Confluence, TestRail, Swagger, Postman, DevTools, Android Studio (Logcat), Charles proxy, Fiddler, Kibana, Grafana, MongoClient
Технологический стек: Java, PostgreSQL, REST API

    Основные обязанности:
  • Тестирование связки справочников
  • Формирование документов отчетности для заинтересованных лиц (.doc .pdf)
  • Сопровождение веб портала и внутреннего продукта
  • Общение с командой аналитиков для актуализации внутренней технической документации на проекте (Confluence, SVN)
  • Доработка и переработка тестовой документации (TestRail) и постановка задач на разработку в Jira
  • Проведение функционального тестирования сервисов (фронтэнд и частично интеграция с бекэндом), проведение регрессионных тестов и приемочных тестов совместно с заказчиком
  • Кросс-браузерное тестирование интерфейса сервисов, проверка верстки и проверка отдельных фичей Android приложения сервиса
  • Работа с базами данных, подготовка тестовых данных для тестовых прогонов, написание SQL запросов средней сложности
  • Тестирование корректности заполнения базы данных, целостности данных и взаимодействия бекэнда с базой
  • Тесное взаимодействие со смежными командами разработки и тестирования для решения актуальных вопросов

Веб-портал туристических бронирований

Длительность проекта: Декабрь 2018 – Март 2021 (2 года 3 месяца)

Сервис по бронированию авиа-билетов, жд-билетов и мест в отелях
Инструменты: GitLab, Jira, Confluence, Figma, Postman, Swagger, DevTools, Android Studio (Logcat), Charles proxy, Fiddler, Kibana, Grafana, Dbeaver, MongoClient, RabbitMQ Admin/span>
Технологический стек: JS, RabbitMQ, MongoDB, PostgreSQL

    Основные обязанности:
  • Проведение функционального и нефункционального тестирования новых реализованных UI форм и компонентов согласно подготовленным спецификациям и дизайнам
  • Работа с базами данных (PostgreSQL, MongoDB), написание SQL запросов для выборки информации, корректировка по результатам обращений, генерации тестовых данных
  • Тестирование логики интеграции внешних сервисов (REST и SOAP) с системой бронирования (Swagger, Postman, SoapUI)
  • Доработка тест кейсов для проведения регресса (TestRail)
  • Работа с моками для имитации поведения сторонних сервисов (Charles)
  • Участие в циклах нагрузочного тестирования, мониторинг состояния системы (Zabbix) и анализ логов приложений (Kibana ELK)
  • Техническая поддержка пользователей веб-сервисов компании (работа с обращениями пользователей, выполнение функционала ServiceDesk, участие в анализе проблем с прода)
  • Тестирование Android – клиента для сервиса
  • Использование внутренних инструментов для настройки и подготовки тестового окружения
  • Аналитика продаж по данным графиков\БД по маркетинговой модели воронки продаж
  • Конфигурация тестового и прод окружения, обеспечение обновлений
  • Проведение тестирования компонентов систем по готовым тест-кейсам и составление баг-репортов
  • Работа в Redmine и Jira
    Достижения:
  • Провёл успешный онбординг двух коллег. Научил использовать внутренние инструменты логирования и аналитики. Консультировал по базам данных (MongoDB/PostgreSQL)
